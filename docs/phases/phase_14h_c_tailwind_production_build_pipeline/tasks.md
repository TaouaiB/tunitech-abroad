# Tasks — Phase 14H-C

## Task 1 — Inspect current static setup
- Read `templates/base.html`.
- Read `config/settings/base.py` and `config/settings/production.py` static settings.
- Check whether `package.json`, `tailwind.config.js`, `static/`, or `assets/` already exist.
- Do not assume paths; inspect first.

## Task 2 — Add Tailwind build files
Create or update:
- `package.json`
- `package-lock.json`
- `tailwind.config.js`
- `static/src/css/app.css`
- `static/css/app.css` generated by build

Recommended `static/src/css/app.css`:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Recommended `tailwind.config.js` content scanning:

```js
module.exports = {
  content: [
    './templates/**/*.html',
    './apps/**/*.py',
    './apps/**/*.html',
  ],
  theme: {
    extend: {},
  },
  plugins: [],
};
```

Adjust paths only if repo structure requires it.

## Task 3 — Replace Tailwind CDN
- Remove Tailwind CDN script from `templates/base.html`.
- Ensure `{% load static %}` is available in base template.
- Add local stylesheet link:

```html
<link rel="stylesheet" href="{% static 'css/app.css' %}">
```

- Keep HTMX/Alpine as-is unless there is a direct conflict.

## Task 4 — Build CSS
Run:

```bash
npm install
npm run css:build
```

Verify `static/css/app.css` exists and is non-empty.

## Task 5 — Add tests/checks
Add a small test, preferably under `apps/core`, that verifies:
- `templates/base.html` does not contain `cdn.tailwindcss.com`
- `templates/base.html` references `{% static 'css/app.css' %}` or the rendered static CSS path
- compiled CSS file exists

Do not over-test Tailwind internals.

## Task 6 — Update docs
Add a short playbook explaining:
- how to install npm dependencies
- how to build CSS once
- how to watch CSS during local UI work
- that `node_modules/` must not be committed
- that collectstatic should run after CSS build during deployment

## Task 7 — Run validation
Run all required checks in `test_plan.md` and `regression_security_checks.md`.
