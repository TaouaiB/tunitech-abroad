# Phase 14K-7 — Final UI QA, Polish Freeze, and Release Readiness

## 1. Purpose

Phase 14K-7 is the final verification and hardening pass for the professional UI/UX polish work completed in Phase 14K-0 through Phase 14K-6.

This phase must make TuniTech Abroad feel consistent, safe, and portfolio-ready without expanding product scope.

This is **not** a new design phase. It is a controlled QA and small-repair phase.

## 2. Operating Mode

Both Gemini and Codex must use this loop:

```text
inspect -> verify -> fix in-scope issues -> enhance only within approved design docs -> test -> repeat until PASS/REPAIRED_PASS
```

Agents must not block for:

- staged vs unstaged files
- mixed working tree
- generated CSS changes
- trailing whitespace
- missing report file
- small copy mismatch
- small route/template mistake that can be fixed without backend changes
- small accessibility issue inside templates

Agents must block only for:

- secret exposure
- CV/privacy leak
- raw provider/LLM payload exposed in templates
- internal integer IDs in public/private links
- forbidden stack or dependency
- backend/model/service/view/URL/settings change required
- new product decision required
- tests failing after reasonable in-scope repair
- any fix that would require Phase 14K scope expansion

## 4. Hard Architecture Rules

The following remain mandatory:

- Django templates + HTMX + Tailwind only.
- No React, Next.js, Vue, Angular, FastAPI, MongoDB, SQLAlchemy, SPA architecture.
- No new frontend dependencies.
- No Tailwind CDN.
- Views stay thin.
- Templates display data only.
- No matching/recommendation logic in templates beyond simple display conditionals.
- No OpenRouter/LLM calls from templates/views.
- No France Travail live API call from user-facing pages.
- Public/user links use UUID `public_id`, not internal integer IDs.
- CV files remain private; never expose `file.url` or raw CV text.
- LLM explanation must not alter fit score.

## 5. QA Areas

### 5.1 Visual consistency

Check that these pages use the same visual language:

- navbar and footer
- cards
- buttons
- badges
- chips
- forms
- empty states
- alerts/messages
- progress bars
- score rings
- legal/error pages

Fix obvious inconsistencies only if template/CSS-only.

### 5.2 Route safety

Verify all `{% url %}` tags point to existing routes.

Special checks:

- no global `{% url 'matching:quick_match' %}` without job `public_id`
- job links use `job.public_id`
- match links use `match.public_id`
- CV status/delete uses `cv.public_id` / `active_cv.public_id`
- no `/dashboard/settings/` invented route
- no fake `/cv-checker/` link unless real route exists
- no `<int:...>` routes introduced

### 5.3 Form and HTMX preservation

Ensure all existing forms preserve:

- method
- action/URL
- CSRF token
- hidden fields
- file `enctype` where needed
- HTMX attributes where already used
- Alpine.js behavior where already used
- validation/error rendering

Do not replace working forms with non-functional links.

### 5.4 Accessibility and responsive review

Fix small template-only issues:

- missing labels
- missing `aria-label` on icon-only controls
- pagination `aria-current`
- progress bars `role="progressbar"` with values
- skip link remains available
- focus styles not removed
- images/icons are decorative or labeled
- dark/light adaptive classes not broken
- mobile layout does not obviously overflow

### 5.5 French UX copy

Normalize obvious copy drift using the approved dictionary.

Canonical labels:

- `Compatibilités`
- `Compétences requises manquantes`
- `Compétences optionnelles à renforcer`
- `Points de vigilance`
- `Test rapide`
- `Bon potentiel`
- `Très bon match`
- `Match partiel`
- `Faible compatibilité`
- `Postuler sur la source`
- `Postuler sur France Travail` only if source is clearly France Travail
- `Source non précisée` if source is missing
- `Entreprise non précisée`
- `Lieu non précisé`
- `Description non disponible`

### 5.6 Privacy and security

Verify no public exposure of:

- CV file URLs
- raw CV text
- raw LLM text/payload except approved explanation text field
- raw France Travail payloads
- secrets or tokens
- internal IDs

## 6. Required Checks

Run the full checks after repairs:

```bash
source .venv/bin/activate

python manage.py check --settings=config.settings.local
python manage.py makemigrations --check --dry-run --settings=config.settings.local
python manage.py test --settings=config.settings.local --parallel 1
npm run css:build
python manage.py collectstatic --dry-run --noinput --settings=config.settings.local
git diff --check
git diff --cached --check || true
```

Run greps:

```bash
grep -R "cdn.tailwindcss.com" templates config static -n || true
grep -R "React\|Next.js\|next/\|createRoot\|vue\|angular\|Vite" package.json templates static apps -n 2>/dev/null || true
grep -R "FranceTravailClient\|search_offers\|api.francetravail" apps/*/admin.py apps/*/views.py templates -n 2>/dev/null || true
grep -R "OpenRouterClient\|OPENROUTER\|run_llm\|client.chat\|_make_request" apps/*/admin.py apps/*/views.py templates -n 2>/dev/null || true
grep -R "file.url\|cv.file.url\|upload.file.url\|raw_text\|raw_request_json\|raw_response_text\|raw_response_json" templates apps -n 2>/dev/null || true
grep -R "<int:" apps templates config -n 2>/dev/null || true
git diff -- . ':!.env' ':!.env.*' | grep -iE "sk-or-|Bearer [A-Za-z0-9_.-]{20,}|OPENROUTER_API_KEY=.*[^=]|FRANCE_TRAVAIL_CLIENT_SECRET=.*[^=]|EMAIL_HOST_PASSWORD=.*[^=]" || echo "OK: no obvious secrets in diff"
```

## 8. Completion Criteria

Phase 14K-7 is complete only when:

- all tests pass
- CSS builds
- collectstatic dry-run passes
- route smoke passes or missing data is clearly explained
- no scope violations
- no backend changes
- no forbidden stack/dependencies
- no secrets
- no CV/raw/provider exposure
- no public internal IDs
- final report exists

Final verdicts:

- `PASS`: no repair needed, all checks pass.
- `REPAIRED_PASS`: in-scope fixes were made and all checks pass.
- `BLOCKED`: only for real issue requiring product/architecture/security decision or scope expansion.
- `FAIL`: serious out-of-scope implementation, privacy leak, forbidden stack, or broken architecture.
